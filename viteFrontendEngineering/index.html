# 前端工程化的痛点（什么是前端工程化）

  • 模块化
    前端的模块化非常杂乱，有 CommonJS， ESModule， CMD， AMD 等等。怎么落实其中的一种规范，并且能够兼容其他的规范，则是前端工程化面对的问题。
  • 兼容浏览器
    怎么让浏览器能够识别我们使用的高级语言（TS 等），语法（箭头函数，const， let 等），高级 API（Promise， Proxy 等）
  • 静态资源
    怎么对图片，svg，字体文件做一个处理
  • 线上产物（打包后的代码）的质量
    怎么对 js，css，图片做一个压缩，自定义分包处理，按需引入，treeShaking 去除无用代码
  • 开发效率
    怎么缩短项目的启动事件，热更新时间

# vite 的解决方案

  • 模块化
    • 选择 ESModule 这种浏览器原生支持的模块化规范（script 标签中 type 属性设置为 module）
    • 利用 ESBuild 来把第三方依赖中的 CommonJS 规范转成 ESModule，从而兼容 CommonJS 规范。
  • 兼容浏览器
    • 利用 ESBuild 来把 TS 转换成 JS（转换前用 tsc 来做 TS 的类型检查）
    • 利用 babal 来将 ES6 的语法转换成 ES5 的代码
    • 利用 core-js 来注入 ES6 后的新 API。
  • 静态资源
    • 配置路径别名从而方便静态资源的导入
    • 利用插件 or loader 来将 SVG 作为组件引入（vite-plugin-vue2-svg 插件（适用于 vue2 项目）, vite-plugin-svgr 插件 (适用于 react 项目), vite-svg-loader（适用于 vue3 项目） ）
    • 内置了对 JSON 文件的解析（底层是使用的 @rollup/pluginutils 的 dataToEsm 方法将 JSON 模块转换成了一个包含各种具名导出的 ES 模块，也可以在 vite 中禁止这种具名导出，直接导出一个对象，从而提高性能）
    • 内置了对媒体类文件（mp4、webm、ogg、mp3、wav、flac 和 aac），字体类文件（woff、woff2、eot、ttf 和 otf），文本类文件 (webmanifest、pdf 和 txt) 的支持，这些文件都可以作为模块导入导出
    • 其他格式的文件可以在 vite 的配置文件里面的配置对象里通过 assetInclude 来让 vite 支持加载
  • 线上产物的质量
    • vite 的代码分割策略，将业务代码，第三方依赖代码，动态导入的代码，静态资源，及其对应的 css 代码打包成单独的 chunk，从而实现拆包，利于缓存，按需引入。并且 vite 的底部使用 rollup 来进行打包，rollup 则提供了自定义拆包策略，让我们可以按照自己的需求进行拆包，提升线上的性能。
    • vite 底部使用的 rollup 进行打包时可以自动 tree shaking 删除无用代码
    • vite 自动对 js，css 代码进行压缩，并且可以用 vite-plugin-imagein 插件来对图片进行压缩
    • 可以使用 vite-plugin-visualizer 插件来进行产物分析，使打包后的产物体积可视化。
  • 开发效率
    • vite 在开发时对业务代码（我们写的代码）直接不打包，no-bundle，并且对于第三方依赖则是使用效率极高的 esbuild 来进行打包，并把 CommonJS 转换成 esmdule，大大缩短了项目启动的时间。
    • 基于原生的 ESModule 规范实现的 HMR API 实现了毫秒级的局部热更新
